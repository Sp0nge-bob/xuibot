# VPN Shop Bot (Platega + 3x-ui)

Полноценный Telegram-бот для продажи VPN-подписок с оплатой через **Platega** и автоматической выдачей ключей через **3x-ui** (главная панель + Nodes).

**Под твой сетап:**
- Подключается **только к главной панели**.
- При покупке создаёт клиентов в нескольких инбаундах (по 1 с каждой ноды) с одним `subId`.
- Одна ссылка на подписку отдаёт все выбранные инбаунды.
- Есть мощный **тестовый режим** — можно полностью протестировать и показать менеджеру Platega без реальных платежей.
- После подключения к Platega переключаешь одной переменной.

---

## 1. Быстрый старт — Тестовый режим на Windows (то, что нужно сейчас)

### Подготовка окружения (один раз)

```powershell
cd $env:USERPROFILE\vpn-platega-bot

python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -U pip
pip install -e .
```

### Настройка .env

```powershell
copy .env.example .env
notepad .env
```

**Минимальные значения для теста:**

```env
BOT_TOKEN=твой_токен
BOT_ADMINS=твой_id

TEST_MODE=true
USE_POLLING=true

# Главная панель (с secret path)
XUI_HOST=https://твоя-главная-панель.com/твой-secret-path/panel/
XUI_USERNAME=admin
XUI_PASSWORD=пароль
# Лучше используй токен (создай в Panel Settings главной панели)
# XUI_TOKEN=твой_токен

# Твоя маскировка подписки
SUBSCRIPTION_BASE_URL=https://example.com/api/v4/GET/

# Инбаунды, которые попадут в подписку (по 1 с каждой ноды)
# Возьми реальные ID из главной панели
DEFAULT_SUBSCRIPTION_INBOUNDS=12,45,67
```

### Запуск

```powershell
.\.venv\Scripts\Activate.ps1
python -m uvicorn app:app --port 8080
```

Зайди в Telegram-бота и нажми `/start`.

В тестовом режиме при выборе тарифа бот:
- Создаст заказ
- В 3x-ui создаст клиентов в указанных инбаундах с общим `subId`
- Сразу выдаст ссылку вида `https://example.com/api/v4/GET/xxxxxxxx` + QR-код

## 2. Как работает тестовый режим

`TEST_MODE=true` полностью симулирует покупку.

- Деньги не списываются.
- Клиенты реально создаются в 3x-ui.
- Выдаётся рабочая подписка по твоей маске.
- Работают "Мои подписки", авто-отключение по времени и т.д.
- Идеально для отладки и показа менеджеру Platega.

Когда будешь готов к реальным платежам:
1. Поставь `TEST_MODE=false`
2. Заполни реальные `PLATEGA_MERCHANT_ID` и `PLATEGA_SECRET`
3. В ЛК Platega укажи Callback URL (публичный HTTPS + `/platega-webhook`)
4. Перезапусти бота

---

## 3. Настройка 3x-ui (важно!)

- Бот работает **только с главной панелью**.
- На нодах подписка должна быть выключена.
- `DEFAULT_SUBSCRIPTION_INBOUNDS` — это те инбаунды (с главной + подключенных нод), которые будут в подписке у каждого покупателя.
- Сделай маску подписки через nginx так, как у тебя сейчас (`/api/v4/GET/`).
- Возьми реальные ID инбаундов из главной панели и укажи их через запятую.

---

## 4. Переход на реальные платежи Platega

1. Получи у менеджера `MerchantId` и `Secret`.
2. В `.env`:
   - `TEST_MODE=false`
   - Заполни `PLATEGA_MERCHANT_ID` и `PLATEGA_SECRET`
3. В ЛК Platega настрой Callback URL на публичный адрес бота + `/platega-webhook` (обязательно HTTPS).
4. Перезапусти бота.

## Структура проекта

- `app.py` — FastAPI (webhook Platega) + запуск бота
- `bot/` — aiogram 3 (меню, покупка, тестовый режим)
- `services/platega.py` — работа с Platega
- `services/xui.py` — создание клиентов и подписок в 3x-ui (поддержка нескольких инбаундов)
- `db/database.py` — SQLite
- `config/` — планы и настройки

---

## Что проверено перед запуском

- **Platega API**: создание транзакции (с методом и без), колбэк с проверкой заголовков `X-MerchantId` + `X-Secret`, обработка статуса `CONFIRMED`.
- **3x-ui 3.3.1 + Nodes**: подключение только к главной панели, создание клиентов в инбаундах с разных нод, один `subId` на всю подписку, корректная сборка ссылки по твоей маске.
- Тестовый режим полностью обходит Platega и выдаёт реальных клиентов.
- Код централизован — одна функция отвечает за выдачу и в тесте, и в проде.
